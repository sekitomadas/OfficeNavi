# -フリーアドレス在籍管理アプリ

## 概要
フリーアドレス製のオフィスにおいて、社員の現在地や在籍状況を共有するための業務アプリケーション

## 背景・課題
フリーアドレス環境では、社員の所在が分かりづらく、
コミュニケーションや業務効率の低下につながることがあります。

本プロジェクトでは、社員が自身の現在位置（席・エリア）を登録し、
他の社員がそれを確認できる仕組みを提供します。

## 想定ユーザー
- 一般社員
- 管理者

## 想定利用シーン
- 社内での在籍確認
- 来客対応時の所在確認

---

## 開発方針

- バックエンドは REST API として先行開発する
- フロントエンドは後続で実装し、APIを共通基盤として利用する
- Controller / Service / Repository の責務を明確に分離する
- View（HTML）を返却しない設計とする
- JSON形式での入出力を前提とする

---

## 開発目的
- フリーアドレス環境における所在把握の課題を解決する
- REST APIを中心とした設計・実装経験を積む
- レイヤードアーキテクチャを意識した設計力を示す
- 転職用ポートフォリオとして、設計意図を説明できる成果物を作成する

---

## 技術選定

### バックエンド
- Java 21（LTS）
- Spring Boot
- Maven
- PostgreSQL

### 採用理由
- **Java 21**
  - LTS（長期サポート）版であり、業務利用を想定した選択
- **Spring Boot**
  - REST API 開発に必要な機能が標準で揃っている
  - 実務での採用実績が高く、再現性のある構成にしやすい
- **PostgreSQL**
  - RDBの基本設計（正規化・制約）を意識した設計が可能
  - 業務システムでの利用実績が多い
- **Maven**
  - 設定の可読性が高く、チーム開発を想定した構成にしやすいため

---

## 機能要件

### 社員管理
- 社員の一覧を取得できる
- 社員の詳細情報を取得できる
- 社員を登録できる

### 座席・エリア管理
- オフィス内のエリア（フロア・ゾーン等）を管理できる
- エリア一覧を取得できる

### 在席情報管理
- 社員が現在の在席エリアを登録できる
- 社員の現在位置を確認できる

---

## 非機能要件

- レスポンス形式はJSONとする
- HTTPステータスコードを適切に返却する
- 入力値チェックを行い、不正なリクエストはエラーとして返却する
- 例外は共通ハンドリングし、レスポンス形式を統一する
- DB アクセスはRepository層に集約する
- ロジックはService層に集約し、Controllerを薄く保つ

---

## MVP（最小構成）

本プロジェクトの MVP は以下とする。

- 社員の登録
- 社員一覧の取得
- 社員の現在位置（エリア）の登録
- 社員の現在位置の取得

※ 認証・認可、履歴管理は MVP 対象外とする
